La Maison Patisserie

Transforming a failing e-commerce experience into a functional, elegant storefront.

Solo UX/UI Designer & Developer · 14 weeks · March–June 2025

Overview

Role

Solo UX/UI Designer & Front-End Developer

Type

Individual project

Duration

14 weeks (March – June 2025)

Tools

Figma, HTML, CSS, JavaScript

Role: End-to-end redesign and development of a broken bakery e-commerce website. Conducted UX audit, rebuilt visual identity, designed responsive system, and developed fully functional front-end.

La Maison Patisserie, a Melbourne French bakery, had a website actively preventing purchases.

Key issues:

  • "Buy Now" buttons leading to 404 errors
  • Illegible pink-on-beige text failing accessibility
  • No form validation
  • Inconsistent visual hierarchy
  • Poor product photography

This was not a cosmetic redesign. It required structural UX correction and full functional rebuild.

Phase 1 — Audit & Research

Heuristic evaluation

Using Nielsen's 10 heuristics, I identified critical violations:

  • No visibility of system status (no cart confirmation)
  • No error prevention (forms accepted invalid data)
  • Inconsistent buttons and typography
  • Poor contrast in error messages
  • Redundant and cluttered layout

The core insight: Fundamental usability failures were blocking transactions.

Competitive review (4 Melbourne bakeries)

I analyzed four local bakery websites to identify patterns of success and failure.

Strong examples balanced: brand warmth, clear hierarchy, accessible contrast, professional imagery, functional checkout flows.

Weak examples either prioritized aesthetics over usability, or function over brand personality.

Insight: Successful bakery sites merge atmosphere with clarity. La Maison had neither consistently.

Design Principles

From research, I established five guiding principles:

  • Harmonious & Accessible
  • Clear Hierarchy
  • Functional First
  • Context-Aware Responsive
  • Prevent Errors

These principles governed every decision.

Visual System Redesign

Problem

  • Decorative Sail font was illegible
  • Pink (#FF6D84) on beige (#F5F2EB) failed contrast
  • Buttons lacked hierarchy
  • Colors felt disconnected

Solution

Typography: Reddit Mono (headings), Poppins (body).

Color palette: Ochre (#CD934B), Burgundy (#8B0214), Soft Black (#0D0907), Warm Beige (#F2EDDF). Burgundy contrast: 8.48:1 (WCAG AAA).

Design intent: French warmth without sacrificing readability.

Button & Interaction System

Original problem: No visual hierarchy between actions.

Solution: Primary — solid ochre, white text, cart icon. Secondary — outline. Exit — burgundy. Design communicates priority immediately.

Error Prevention & Form Validation

Original issue: Forms accepted invalid card numbers and empty fields.

Solution: Format placeholders, character limits, inline validation, clear error messaging, required indicators.

Goal: Prevent errors before submission.

Responsive Redesign

Mobile redesign was not just scaled-down desktop.

Key changes: Horizontal nav → full-screen hamburger. Two-column cart → single column. Always-visible order summary → collapsible accordion. Hero overlay → full-bleed layout. Each breakpoint optimized for usage context.

Communicating Constraints

La Maison offers pickup only. Original site communicated this late.

Solution: Prominent pickup warning in cart with: attention icon, location and time details, clickable date/time modification. Prevents late-stage abandonment.

Development

Built fully functional front-end using HTML5, CSS3, vanilla JavaScript.

Structured codebase: /assets, /styles, /scripts.

Implemented: Interactive cart with real-time totals; session persistence; validation logic; product modal overlay; responsive navigation.

Before / After Impact

Delivered improvements in: visual clarity, functional reliability, accessibility compliance, error prevention, navigation logic.

All buttons functional. All forms validated. Clear hierarchy across screens.

Reflection

  • Fundamentals compound: spacing, hierarchy, contrast.
  • Clear communication reduces friction.
  • Design systems simplify development.
  • Responsive design adapts to behavior, not width.
  • Development tools accelerate work but require understanding.
